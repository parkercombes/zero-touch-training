Design decisions reflected in this mock (see docs/design-decisions.md)

  • FAB (floating action button) carries the same badge state as the launchpad tile — amber "1" here means Level 2 in progress. Moving the FAB mirrors the panel as it opens (right position shifts left by panel width).
  • Panel defaults to Help Now mode when opened from inside an app (DDR-006: context-driven default). The step list shows the user exactly where they are in the real transaction, with the active step pulsing.
  • Step callout bubble appears over the highlighted field in the SAP form simultaneously — both the panel step and the field highlight update together as the user progresses.
  • App content compresses when panel opens (margin-right = panel width). No overlay/dim — the user should be able to read the real SAP fields while following the steps.
  • Practice mode in the panel shows the 4-level card stack — the user can jump to any completed level or resume from the current one.
  • "Open full trainer →" in the footer escalates to the full-page standalone trainer if the panel is too narrow for complex steps.
  • DDR-003 (FAB vs. header icon): FAB used here. Note that it sits above any SAP footer bar — check for footer conflicts in production SAP environments.
  • DDR-004 (side panel vs. bottom drawer): Right slide-in shown here. At <768 px viewport, this panel should switch to a bottom drawer.
